# Химические данные: виды, представления и особенности

**Данные** — формализованное представление информации, которое можно интерпретировать и обрабатывать.

**Типы данных:**

- **Структурированные** — таблицы с заранее заданной схемой.
    
    - Пример: таблица свойств молекул в PubChem.
        
- **Полуструктурированные** — нет жесткой табличной схемы, но есть иерархия и метаданные.
    
    - Пример: HTML, JSON, XML, CSV.
        
- **Неструктурированные** — нет заранее заданной схемы.
    
    - Пример: статьи, патенты, изображения, видео, PDF.
        

# Представление химических объектов

Химический объект можно представить разными способами:

- химическая формула;
    
- структурная формула;
    
- SMILES;
    
- InChI;
    
- молекулярный граф;
    
- 3D-структура;
    
- дескрипторы;
    
- фингерпринты;
    
- спектры;
    
- текстовое описание.
    

# SMILES

**SMILES** — популярный строковый формат для представления органических молекул.

Особенности:

- одна и та же молекула может иметь несколько разных SMILES;
    
- перед удалением дубликатов SMILES обычно **канонизируют**;
    
- канонический SMILES нужен, чтобы разные записи одной молекулы привести к одному виду;
    
- SMILES часто используют как вход для построения молекулярного графа.
    

Пример идеи:

```text
Одна молекула → разные SMILES → канонизация → единая запись
```

# InChI

**InChI** — идентификатор, который однозначно отражает структуру молекулы.

Особенности:

- удобен для поиска, хранения и сопоставления записей в базах данных;
    
- в ML-моделях используется реже, чем SMILES, графы или фингерпринты;
    
- часто нужен как стабильный идентификатор вещества.
    

# Молекулярные графы

Молекулу можно представить как граф:

- **атомы** → вершины графа;
    
- **химические связи** → ребра графа.
    

Такое представление удобно для моделей, которые работают со структурой молекулы напрямую.

# 3D-структура

Некоторые свойства молекул зависят не только от состава, но и от пространственной организации.

Важно для задач:

- связывание с белком;
    
- каталитическая активность;
    
- межмолекулярные взаимодействия;
    
- моделирование конформаций.
    

Форматы хранения 3D-структур:

- PDB;
    
- MOL2;
    
- SDF;
    
- XYZ.
    

# ML representation

Для машинного обучения химический объект нужно превратить в числовое представление.

Основные варианты:

- **дескрипторы**;
    
- **фингерпринты**;
    
- **молекулярные графы**;
    
- **эмбеддинги**;
    
- **3D-признаки**.
    

# Дескрипторы

**Дескрипторы** — числовые признаки, описывающие свойства молекулы.

Примеры дескрипторов:

- molecular weight — молекулярная масса;
    
- LogP — липофильность;
    
- topological polar surface area;
    
- hydrogen bond donors;
    
- hydrogen bond acceptors;
    
- rotatable bonds;
    
- aromatic rings;
    
- heavy atom count;
    
- formal charge.
    

Дескрипторы превращают молекулу в обычный числовой вектор, который можно подать в ML-модель.

# Фингерпринты

**Фингерпринт** — компактный вектор, где каждый элемент показывает наличие или отсутствие структурного признака.

Примеры признаков:

- ароматическое кольцо;
    
- карбоксильная группа;
    
- амин;
    
- конкретный фрагмент атомного окружения.
    

Частый пример:

```text
Morgan Fingerprint → бинарный вектор из 0 и 1
```

Где:

- `1` — признак присутствует;
    
- `0` — признак отсутствует.
    

# Особенности химических данных

Для химических данных характерны:

- ограниченный объем;
    
- экспериментальный шум;
    
- ценные выбросы;
    
- дисбаланс классов;
    
- неоднородность источников.
    

# Ограниченный объем данных

В химии данные часто нельзя просто собрать автоматически в огромном количестве.

Причина:

- многие значения требуют реального эксперимента;
    
- эксперимент может быть дорогим, долгим или сложным;
    
- не все результаты публикуются;
    
- часть данных остается в закрытых лабораторных системах.
    

# Экспериментальный шум

Одна и та же молекула может давать разные измерения в разных условиях.

Причины шума:

- разное оборудование;
    
- разное качество реагентов;
    
- разные клеточные линии;
    
- разные протоколы измерений;
    
- человеческий фактор.
    

Поэтому важно хранить не только значение, но и контекст измерения:

- метод;
    
- единицы измерения;
    
- условия эксперимента;
    
- источник;
    
- тип assay;
    
- организм / клеточная линия;
    
- температура, pH, растворитель и т.д.
    

# Выбросы

В химии выброс не всегда ошибка.

Возможные причины выброса:

- ошибка измерения;
    
- ошибка разметки;
    
- необычное, но реальное свойство вещества;
    
- редкий механизм действия;
    
- отличия в условиях эксперимента.
    

Удалять выбросы автоматически опасно: можно потерять наиболее ценные соединения.

# Дисбаланс классов

В химических датасетах часто много данных одного типа и мало данных другого.

Пример:

- много неактивных соединений;
    
- мало активных соединений.
    

Или наоборот:

- в статьях чаще публикуют успешные результаты;
    
- отрицательные результаты часто не попадают в датасеты.
    

Проблема:

- модель видит неполную картину;
    
- модель может переобучиться на «успешные» случаи;
    
- качество на реальных данных может быть хуже, чем на тестовом наборе.
    

# Null result box

**Null result box** — проблема отсутствия отрицательных результатов.

Суть:

- статьи чаще публикуют положительные и интересные результаты;
    
- неудачные эксперименты часто не публикуются;
    
- модель обучается на том, что «сработало»;
    
- модель хуже понимает, что «не работает».
    

Для химического ML это критично, потому что отрицательные примеры помогают отделять перспективные молекулы от неперспективных.

# Неоднородность источников

Химические данные редко приходят из одного источника.

Источники могут различаться по:

- формату;
    
- единицам измерения;
    
- протоколам;
    
- качеству;
    
- уровню обработки;
    
- полноте метаданных.
    

Перед анализом данные нужно привести к единому виду.

# Нормализация химических данных

Что обычно нужно нормализовать:

- названия веществ;
    
- SMILES;
    
- InChI / InChIKey;
    
- CAS;
    
- единицы измерения;
    
- числовые значения;
    
- условия эксперимента;
    
- названия мишеней;
    
- типы assay;
    
- источники данных.
    

Пример пайплайна:

```text
Разные источники
    ↓
Извлечение данных
    ↓
Очистка
    ↓
Канонизация структур
    ↓
Нормализация единиц
    ↓
Удаление или объединение дубликатов
    ↓
Проверка качества
    ↓
ML-ready dataset
```

# Главное

Химические данные — это не просто строки в таблице.

За каждой записью могут стоять:

- структура молекулы;
    
- способ представления;
    
- экспериментальный метод;
    
- условия измерения;
    
- источник;
    
- шум;
    
- возможные ошибки;
    
- неполнота данных.
    

Поэтому перед ML важно не только собрать данные, но и привести их к корректному химическому и машинно-читаемому виду.


# Где искать химические данные

Основные источники химических данных:

- открытые базы данных;
    
- базы данных расчетной химии;
    
- опубликованные статьи;
    
- собственные эксперименты;
    
- высокопроизводительный скрининг — HTS.
    

# Открытые химические базы данных

**PubChem**

Крупная открытая база химической информации.

Что можно искать:

- вещества по названию;
    
- молекулярную формулу;
    
- структуру;
    
- идентификаторы;
    
- физико-химические свойства;
    
- биологическую активность;
    
- токсичность и безопасность;
    
- патенты;
    
- ссылки на литературу.
    

Для Python есть **PubChemPy API**.

Что позволяет делать:

- искать вещества по названию;
    
- искать по субструктуре;
    
- искать по сходству;
    
- получать химические свойства;
    
- преобразовывать форматы;
    
- выполнять химическую стандартизацию.
    

**ChEMBL**

База биологически активных молекул с лекарственными свойствами.

Содержит:

- химические данные;
    
- биоактивность;
    
- геномные данные;
    
- данные по мишеням и assay.
    

Полезна для задач drug discovery и QSAR.

**DrugBank**

База данных по лекарствам и лекарственным продуктам.

Содержит:

- лекарственные взаимодействия;
    
- фармакологию;
    
- химические структуры;
    
- мишени;
    
- метаболизм;
    
- данные для drug repurposing;
    
- данные для ML-моделей.
    

**STITCH / STRING**

Базы для анализа взаимодействий.

STITCH:

- сети химико-белковых взаимодействий;
    
- связи между соединениями и белками.
    

STRING API:

- программный доступ к данным;
    
- удобно использовать без ручной работы через сайт.
    

**ChemSpider**

Бесплатная база химических структур.

Поддерживает поиск:

- по структуре;
    
- через рисование структуры на сайте;
    
- через загрузку структурного файла;
    
- по данным из разных источников.
    

**Кристаллографические базы**

Используются для поиска кристаллических структур.

Могут содержать:

- органические структуры;
    
- неорганические структуры;
    
- металлоорганические структуры;
    
- минералы.
    

Обычно не включают биополимеры.

# Базы данных расчетной химии

**Materials Project**

База данных для материаловедения.

Содержит:

- структуры материалов;
    
- молекулы;
    
- элементы;
    
- фазовые диаграммы;
    
- данные по батареям;
    
- данные по электродам;
    
- расчетные свойства.
    

Есть API для программного получения данных.

**DFT-базы данных**

Содержат расчетные свойства материалов.

Примеры данных:

- термодинамические свойства;
    
- структурные свойства;
    
- энергия;
    
- стабильность;
    
- расстояние до convex hull;
    
- пространственная группа.
    

Такие базы полезны, когда экспериментальных данных мало или их дорого получать.

# Как работать с базами данных

Базовый порядок:

```text
Понять задачу
    ↓
Выбрать подходящую базу
    ↓
Проверить доступность данных
    ↓
Получить данные через API или парсинг
    ↓
Очистить и нормализовать
    ↓
Собрать ML-ready dataset
```

# API и инструменты

Популярные инструменты в химии и материаловедении:

- **RDKit** — работа с молекулами, SMILES, дескрипторами, фингерпринтами;
    
- **Open Babel** — конвертация химических форматов;
    
- **PySCF** — квантово-химические расчеты;
    
- **Avogadro** — визуализация и работа со структурами;
    
- **Reaxys API** — доступ к химическим данным Reaxys;
    
- **NIST Materials Data Repository** — данные по материалам.
    

# Парсинг химических данных

Если API нет или его недостаточно, используют парсинг.

Инструменты:

- **Requests** — загрузка страниц и API-ответов;
    
- **BeautifulSoup** — парсинг HTML;
    
- **LXML** — быстрый парсинг HTML/XML;
    
- **Scrapy** — массовый веб-скрейпинг;
    
- **Selenium** — сайты с JavaScript;
    
- **PyQuery** — jQuery-подобный парсинг HTML;
    
- **MechanicalSoup** — формы и простая автоматизация браузера.
    

# Где брать экспериментальные данные

Основные варианты:

- опубликованные статьи;
    
- собственный эксперимент;
    
- высокопроизводительный скрининг — HTS.
    

**Статьи** удобны, если данные уже опубликованы, но их часто нужно вручную извлекать из текста, таблиц, графиков и supplementary materials.

**Эксперимент** дает самые релевантные данные, но требует времени, реагентов, оборудования и контроля условий.

**HTS** позволяет быстро проверить большое число соединений, но требует инфраструктуры и аккуратной обработки результатов.

# Работа со статьями

Перед сбором данных из статей нужно:

- определить целевую величину;
    
- выбрать параметры, которые могут с ней коррелировать;
    
- сначала собрать широкий список признаков;
    
- заранее выбрать формат хранения;
    
- фиксировать источник каждой записи;
    
- сохранять условия эксперимента;
    
- быть готовым к медленному ручному процессу.
    

# Где искать статьи

Основные источники:

- Google Scholar;
    
- ScienceDirect;
    
- Semantic Scholar;
    
- ResearchRabbit;
    
- сайты журналов;
    
- библиографические ссылки внутри статей.
    

ResearchRabbit и Semantic Scholar полезны для поиска связанных работ и построения сети статей.

# Как оценивать журналы

При выборе источника можно смотреть:

- квартиль журнала — Q1, Q2, Q3, Q4;
    
- SJR;
    
- Impact Factor;
    
- релевантность журналу и теме;
    
- тип статьи;
    
- наличие supplementary materials.
    

Высокий рейтинг журнала не гарантирует, что данные подойдут для ML. Важнее полнота, структура и воспроизводимость данных.

# Как получить статью

Варианты:

- официальный доступ через университет или подписку;
    
- Google Scholar;
    
- ResearchGate;
    
- open access версия;
    
- supplementary materials на сайте журнала.
    

# Типы статей

**Data article**

Статья, основная цель которой — описать научно ценный набор данных.

Примеры журналов:

- Scientific Data;
    
- Data in Brief;
    
- Journal of Chemical and Engineering Data.
    

**Review article**

Обзорная статья.

Полезна для:

- понимания области;
    
- поиска ключевых терминов;
    
- поиска основных датасетов;
    
- поиска ссылок на первичные исследования.
    

Примеры журналов:

- Nature Reviews Chemistry;
    
- Chemical Reviews;
    
- Digital Discovery.
    

**Research article**

Исследовательская статья.

Содержит:

- постановку задачи;
    
- методы;
    
- эксперимент;
    
- результаты;
    
- интерпретацию;
    
- сравнение с другими работами;
    
- supplementary materials.
    

# Как читать исследовательскую статью для сбора данных

Полезные части статьи:

- **Abstract** — быстро понять задачу и объект исследования;
    
- **Introduction** — понять контекст и термины;
    
- **Materials and Methods** — достать условия эксперимента;
    
- **Results** — извлечь численные значения;
    
- **Discussion** — понять интерпретацию;
    
- **Supplementary Information** — найти дополнительные таблицы, рисунки и параметры.
    

# Supplementary materials

Дополнительные материалы часто важнее основного текста для датасета.

Там могут быть:

- полные таблицы;
    
- дополнительные графики;
    
- параметры эксперимента;
    
- исходные данные;
    
- методики;
    
- данные, которые не поместились в статью.
    

При сборе датасета supplementary materials нужно проверять обязательно.

# Статья-агрегатор

Иногда данные уже собраны в статье-агрегаторе.

Пример из презентации:

- 48 000 материалов;
    
- металлы и полупроводники;
    
- данные доступны в файлах и через Materials Project.
    

Плюс такого источника: данные уже собраны и частично структурированы.

Минус: нужно проверять, как именно они были получены и нормализованы.

# Извлечение данных из графиков

Иногда нужные значения не указаны в тексте или таблицах, а есть только на графике.

Тогда используют вспомогательные инструменты:

- извлечение данных из графиков;
    
- ручная разметка точек;
    
- восстановление значений по осям;
    
- проверка единиц измерения.
    

Важно: такие данные менее надежны, чем исходная таблица.

# Хранение и упорядочивание статей

Для работы со статьями нужен менеджер ссылок.

Пример: **Mendeley**.

Зачем нужен:

- хранить PDF;
    
- систематизировать статьи;
    
- делать заметки;
    
- создавать библиографию;
    
- импортировать документы;
    
- делиться статьями с командой;
    
- находить похожие статьи.

# Главное

Для химического ML важно не просто найти данные, а понять их происхождение.

Нужно проверять:

- откуда взяты данные;
    
- экспериментальные они или расчетные;
    
- есть ли условия измерения;
